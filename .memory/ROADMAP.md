# Roadmap — V2 (драйверы провайдеров)

## Цели
- Зафиксировать разработку версии 2.
- Собрать полный контекст для драйверов `gemini-3-pro-image-preview` и `gpt-image-1.5-2025-12-16`.
- Подготовить план интеграции этих драйверов в проект.

## Контекст для внедрения драйверов (сбор)
1. API‑документация по image‑эндпоинтам и формату requests/responses.
2. Формат передачи изображения (inline/base64) и порядок частей.
3. Формат ответа и извлечение image‑payload.
4. Ошибки и retryable‑коды (transient), наличие аналога `NO_IMAGE`.
5. Лимиты: размер изображения, RPM/TPM, типы MIME.
6. Аутентификация (ключи, заголовки).
7. Поддерживаемые параметры модели (model id, output mime).
8. Минимальные тестовые примеры: успешный и ошибочный ответ.

## План интеграции (outline)
1. Обновить `slot.settings` схемы под новые модели (минимальный набор полей).
2. Реализовать драйверы с inline‑image, возвращающие `ProviderResult`.
3. Унаследовать ретраи от GeminiDriver (retry_policy + NO_IMAGE), если применимо.
4. Обновить provider docs с параметрами, ретраями, логированием.
5. Добавить/расширить unit‑тесты драйверов (success/timeout/error/invalid_response).
6. Проработать UI‑опцию выбора формата/размера результата, если провайдер поддерживает такую настройку.
